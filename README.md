How are graceful degration and service workers related?
- Graceful degration is the concept of slowly reducing functionality instead of a complete crash. When certain features are down, you substitute it with another feature rather than just completely shutting down. Service workers are tools that will help us have more control over what our app is able to do when there is no internet connection. 
- Graceful degration and service workers are related in the sense that when there is no internet connection, or someone's on airplane mode, or are in the middle of a forest, we reduce the functionality of our website/code but don't completely halt it.  Using service workers for graceful degradation means our code falls back to a simipler, yet still effective UI when the network is unstable, ensuring the user retains access to core content even when there's no connection.
